Tags are short, freeform labels you attach to notes to create connections that cut across notebook boundaries. While every note belongs to exactly one notebook, a single note can carry as many tags as you like, and a single tag can appear on notes scattered across dozens of notebooks. This makes tags ideal for capturing the *nature* or *status* of a note (an idea, a to-do, a meeting record) independent of the topic or project it belongs to.

## Adding Tags to a Note

<Steps>
  <Step title="Open the note">
    Click any note in the note list to open it in the editor.
  </Step>
  <Step title="Click the Tags field">
    Below the note title, click the **Tags** field. It reads *Add a tag…* when empty.
  </Step>
  <Step title="Type your tag and press Enter">
    Type the tag name (for example, `meeting`) and press <kbd>Enter</kbd>. The tag is created if it does not already exist, and it is applied to the note immediately. Repeat to add multiple tags.
  </Step>
  <Step title="Remove a tag">
    Click the **×** next to any tag pill in the Tags field to remove it from the note. The tag itself is not deleted — it remains available for other notes.
  </Step>
</Steps>

## Filtering Notes by Tag

Once you have tagged a few notes, you can surface them instantly without running a search.

**Sidebar tag list** — The sidebar contains a **Tags** section that lists every tag you have used. Click any tag name to see all notes carrying that tag, regardless of which notebook they live in.

**Tag cloud** — Open the **Tag Cloud** view from the sidebar footer to see all your tags sized by frequency. Click any tag in the cloud to filter the note list to matching notes.

**Combining filters** — You can click a notebook in the sidebar *and* a tag at the same time to see only notes that match both criteria. This is equivalent to using the `notebook:` and `tag:` search operators described in [Search](/concepts/search).

## Renaming and Deleting Tags

Keeping your tag vocabulary tidy over time is straightforward.

<Accordion title="Rename a tag">
  1. Go to **Settings → Tags** (or right-click the tag name in the sidebar and select **Rename**).
  2. Click the pencil icon next to the tag you want to rename.
  3. Type the new name and press <kbd>Enter</kbd>.

  Every note that previously carried the old tag name is updated to use the new name automatically.
</Accordion>

<Accordion title="Delete a tag">
  1. Go to **Settings → Tags** (or right-click the tag name in the sidebar and select **Delete**).
  2. Click the trash icon next to the tag and confirm in the dialog that appears.

  Deleting a tag removes it from every note it was applied to. The notes themselves are not affected.
</Accordion>

## Best Practices

A consistent tagging strategy pays dividends as your knowledge base grows. A few guidelines:

- **Keep tags short** — one or two words at most. `meeting` is better than `meeting-notes-from-work`.
- **Use lowercase consistently** — tags are case-insensitive, but sticking to lowercase avoids accidental duplicates like `Todo` and `todo`.
- **Prefer a small, stable vocabulary** — a focused set of tags you reuse is far more useful than a unique tag per note.
- **Useful starter tags:** `#meeting`, `#idea`, `#todo`, `#done`, `#review`, `#reference`, `#resource`

<Tip>
  Use tags to track **status** (`#todo`, `#done`, `#review`) and notebooks to track **topics or projects** (Work, Learning, Side Projects). Mixing the two roles leads to clutter — keeping them separate makes both systems more powerful.
</Tip>
